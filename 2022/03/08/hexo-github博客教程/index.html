  4. 输入命令

Hexo

官方网址：https://hexo.io/zh-cn/index.html

输入命令

检查：

输入命令

二、搭建GitHub仓库

注册GitHub账号

GitHub账号注册教程：https://www.bilibili.com/read/cv5107169

创建仓库

Repository name：“GitHub用户名“+.github.io

选择Public勾选Add a README file点击Create repository

博客地址即为：“用户名“+.github.io

三、 生成SSH Keys

打开任意一个文件夹点击鼠标右键选择Git Bash Here输入命令SSH

检查电脑是否安装SSH

输入命令ssh-keygen -t rsa -C "邮箱地址"

点击Enter键4次

此时会在C盘【C:\Users\用户名（每个人不一样）.ssh】生成.pub文件

打开该文件复制其中所有内容

打开GitHub点击用户头像点击Sittings进入页面左侧有SSH and GPG keys点击进入New SSH key将该复制的.pub内的所有内容粘贴进入。

返回前一步骤的Git Bash Here命令行输入

若出现

Hi “用户名“! You’ve successfully authenticated, but GitHub does not provide shell access.

则生成SSH Keys已经成功！

四、 生成本地博客内容

在喜欢的盘中新建一个文件夹，打开之后右键点击Git bash here

  • 输入命令hexo init

等待，直到输出Start blogging with Hexo!

  • 输入命令Hexo g

  • 输入命令Hexo s

五、生成网络博客内容

打开_config.yml文件，配置文件修改

deploy:
type: git
repository: https://github.com/你的名字/仓库的名字.git
branch: main

在博客文件页面右键选择git bash here安装hexo-deployer-git自动部署发布工具:npm install hexo-deployer-git --save

输入命令Hexo g

输入命令hexo d

如果输入 hexo d之后出现以下错误：

fatal: unable to access ‘https://github.com/lixiaoyao-web/lixiaoyao-web.github.io.git/': SSL_write() returned SYSCALL, errno = 49

FATAL {

err: Error: Spawn failed

at ChildProcess. (/Users/apple/blog/node_modules/hexo-util/lib/spawn.js:51:21)

at ChildProcess.emit (events.js:315:20)

at Process.ChildProcess._handle.onexit (internal/child_process.js:275:12) {

code: 128

}

} Something’s wrong. Maybe you can find the solution here: %s https://hexo.io/docs/troubleshooting.html

则先输入：

git config –global user.email “你的邮箱”

则再输入：

git config –global user.name “你的gihub名字”

然后

hexo clean

hexo g

hexo d

应该就不会出错了！如果hexo d 之后不动，直接按ctl+c终止，之后再hexo d 就会开始上传！

六、美化

使用css/js引入的方法

在themes\fluid\source\css\文件夹下新建一个文本文档，找到一段可用的css美化代码，复制粘贴的新建的文档里面，然后把文档的**.txt修改成.css/.js**，命好名字。

然后打开主题配置文件**/themes/fluid/_config.yml**找到“custom_js”和“custom_js”下面进行引入。
